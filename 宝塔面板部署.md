# WatchDog — 宝塔面板部署教程

> 本文档适用于通过宝塔（aaPanel）部署 WatchDog 监控平台的用户。
> 推荐系统：Ubuntu 20.04 / 22.04 LTS 或 CentOS 7/8。

---

## 前置要求

| 组件 | 版本 | 说明 |
|------|------|------|
| 宝塔面板 | 7.x / 8.x | https://www.bt.cn |
| Nginx | 1.20+ | 推荐宝塔一键安装 |
| PHP | 8.1+ | 需开启 `zip`, `curl`, `mbstring`, `pdo_mysql` |
| MySQL | 5.7+ / 8.0 | 推荐 MySQL 8.0 |
| Node.js | 18+ | 用于 ws-relay 中继服务 |
| WordPress | 6.x | 部署在已配置的 PHP 站点 |

---

## 第一步：安装宝塔必备软件

登录宝塔面板 → **软件商店**，安装以下组件：

1. **Nginx**（推荐 1.22）
2. **MySQL 8.0**
3. **PHP 8.2**（安装后进入 PHP 管理 → 安装扩展：`fileinfo`, `curl`, `mbstring`, `opcache`）
4. **PM2管理器**（宝塔版 Node.js 进程守护）

---

## 第二步：创建网站和数据库

### 2.1 创建站点

宝塔 → **网站** → **添加站点**：

| 字段 | 填写 |
|------|------|
| 域名 | `your-domain.com`（或服务器 IP） |
| 根目录 | `/www/wwwroot/watchdog` |
| PHP 版本 | 8.2 |
| 数据库 | 创建同名数据库，记录用户名/密码 |

### 2.2 开启 HTTPS（推荐）

网站 → 选择刚创建的站点 → **SSL** → 申请免费证书（Let's Encrypt）→ 开启强制 HTTPS。

---

## 第三步：部署 WordPress

```bash
# 进入站点目录
cd /www/wwwroot/watchdog

# 下载 WordPress
wget https://wordpress.org/latest.zip
unzip latest.zip
mv wordpress/* .
rm -rf wordpress latest.zip
```

在宝塔 **文件管理** 中找到 `wp-config-sample.php`，复制为 `wp-config.php`，填入第 2.1 步的数据库信息：

```php
define('DB_NAME', 'watchdog');
define('DB_USER', 'watchdog_user');
define('DB_PASSWORD', 'your_password');
define('DB_HOST', '127.0.0.1');
define('DB_CHARSET', 'utf8mb4');
```

浏览器访问域名，完成 WordPress 安装向导（管理员账号记好）。

---

## 第四步：安装 WatchDog 插件或主题

### 方式 A — 插件版

```bash
# 将 wordpress-plugin 文件夹上传到 WordPress 插件目录
cp -r /path/to/watchdog/wordpress-plugin /www/wwwroot/watchdog/wp-content/plugins/watchdog-monitor
```

WordPress 后台 → **插件** → 找到 **WatchDog Monitor** → 启用。

### 方式 B — 主题版（推荐）

```bash
# 将 wordpress-theme 文件夹上传到 WordPress 主题目录
cp -r /path/to/watchdog/wordpress-theme /www/wwwroot/watchdog/wp-content/themes/watchdog-monitor
```

WordPress 后台 → **外观** → **主题** → 找到 **WatchDog Monitor** → 启用。

主题激活后会自动创建所需数据库表和页面。访问 `https://your-domain.com/wd-dashboard` 登录。

---

## 第五步：部署 WebSocket 中继服务

### 5.1 安装 Node.js

宝塔 → **软件商店** → 搜索 **Node.js** → 安装版本 18.x。

### 5.2 安装依赖并配置

```bash
cd /www/wwwroot/watchdog-ws   # 新建目录，放 ws-relay 文件

# 上传 ws-relay 目录内容到此处
cp -r /path/to/watchdog/ws-relay/* /www/wwwroot/watchdog-ws/

# 安装依赖
cd /www/wwwroot/watchdog-ws
npm install

# 创建环境配置
cp .env.example .env
```

编辑 `.env`（**只需填两个变量**）：

```env
# WebSocket 监听端口，与 Nginx proxy_pass 端口保持一致
WS_PORT=8765

# 与 WordPress wp_options.watchdog_ws_secret 相同的值
# 获取方法见下方说明
WS_SECRET=从数据库读取后填入
```

> **如何获取 `WS_SECRET`**
>
> WatchDog 插件/主题**首次运行**时会自动在数据库生成该密钥。
> 在宝塔 **数据库** → phpMyAdmin（或命令行）执行：
>
> ```sql
> SELECT option_value FROM wp_options WHERE option_name = 'watchdog_ws_secret';
> ```
>
> 将查询结果（一串 64 位十六进制字符串）填入 `.env` 的 `WS_SECRET`。
>
> **如果想自己预设密钥**，可先生成再写入数据库：
>
> ```bash
> # 生成 32 字节随机字符串
> node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
> ```
>
> 将输出值填入 `.env`，再在数据库执行：
>
> ```sql
> INSERT INTO wp_options (option_name, option_value, autoload)
> VALUES ('watchdog_ws_secret', '你生成的值', 'yes')
> ON DUPLICATE KEY UPDATE option_value = '你生成的值';
> ```

### 5.3 用 PM2 管理器启动

宝塔 → **PM2管理器** → **添加项目**：

| 字段 | 填写 |
|------|------|
| 名称 | `watchdog-ws` |
| 文件路径 | `/www/wwwroot/watchdog-ws/server.js` |
| 运行目录 | `/www/wwwroot/watchdog-ws` |
| 启动参数 | `--max-memory-restart 200M` |

点击「**开启**」，状态变为 `online` 即成功。

---

## 第六步：配置 Nginx 反向代理（WebSocket）

宝塔 → 网站 → 选择主站 → **配置文件**，在 `server {}` 块内追加：

```nginx
# WebSocket 反向代理（将 /ws/ 路径转发给 Node.js）
location /ws/ {
    proxy_pass         http://127.0.0.1:8765/;
    proxy_http_version 1.1;
    proxy_set_header   Upgrade    $http_upgrade;
    proxy_set_header   Connection "upgrade";
    proxy_set_header   Host       $host;
    proxy_read_timeout 3600s;
    proxy_send_timeout 3600s;
}
```

> **配置后 WebSocket 地址**：`wss://your-domain.com/ws`
> 若不配置反向代理，WebSocket 地址为：`ws://your-server-ip:8765`

---

## 第七步：开放防火墙端口

宝塔 → **安全** → 防火墙，**开放端口**：

| 端口 | 协议 | 用途 |
|------|------|------|
| 80   | TCP  | HTTP |
| 443  | TCP  | HTTPS |
| 8765 | TCP  | WebSocket 中继（若未用 Nginx 代理则需开放） |

同时确认云服务器安全组（阿里云/腾讯云/AWS）也放开上述端口。

---

## 第八步：在 WatchDog 后台配置接口

1. 访问 WatchDog 后台 → **KOOK 机器人** 页面  
2. 填写 **WebSocket 中继地址**：`your-server-ip:8765`（或 `your-domain.com/ws`）
3. 保存

---

## 第九步：构建 Windows 客户端

在本地 Windows 机器上运行 WatchDog Generator：

```powershell
cd generator
pip install -r requirements.txt
python main.py
```

在生成器中填入：

| 字段 | 值 |
|------|----|
| REST API 地址 | `https://your-domain.com/wp-json/watchdog/v1` |
| API Key | 在 WatchDog 后台 → API 管理 → 创建并复制 |
| WebSocket 地址 | `ws://your-server-ip:8765` 或 `wss://your-domain.com/ws` |

点击「**打包输出**」→「**开始构建**」，生成 EXE 后部署到目标 Windows 机器。

---

## 验证部署

访问以下地址，应看到对应结果：

| 地址 | 预期结果 |
|------|---------|
| `https://your-domain.com/wd-dashboard` | WatchDog 登录页 |
| `https://your-domain.com/wp-json/watchdog/v1/heartbeat` | HTTP 401（端点存在） |
| `http://your-server-ip:8765/health` | `{"status":"ok",...}` |
| `http://your-server-ip:8765/ping` | `pong` |

在 WatchDog 后台 → API 管理 → **接口自检**，确认所有项目为绿色。

## 常见问题

### WordPress 固定链接报 404

宝塔 → 网站 → 配置文件，在 `location /` 内加入 WordPress 伪静态规则：

```nginx
location / {
    try_files $uri $uri/ /index.php?$args;
}
```

### WebSocket 连接失败

1. 确认 PM2 进程状态为 `online`（宝塔 PM2管理器查看）
2. 确认防火墙已开放 8765 端口
3. 宝塔日志中检查 ws-relay 日志

### PHP 内存不足

宝塔 → PHP → 配置修改，调整 `memory_limit = 256M`。

### Node.js 版本不对

宝塔 PM2管理器 → 版本管理 → 安装 Node 18.x，切换版本后重启项目。

## 文件权限设置

```bash
# 设置 WordPress 目录权限
chown -R www:www /www/wwwroot/watchdog
find /www/wwwroot/watchdog -type d -exec chmod 755 {} \;
find /www/wwwroot/watchdog -type f -exec chmod 644 {} \;
chmod 600 /www/wwwroot/watchdog/wp-config.php
```
