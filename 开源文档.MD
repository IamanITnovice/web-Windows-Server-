## 一、这个项目是干嘛的

WatchDog Monitor 是一个跑在浏览器里的 Windows 远程管理系统。它形式上是一个 WordPress 主题，但实际干的活是管理 Windows 服务器。

说白了就是：你把主题装到 WordPress 上，激活，配好，然后打开浏览器就能管你的 Windows 机器。不用装 TeamViewer，不用开 RDP，不用折腾什么 VPN。点开网页就行。

这项目最开始是因为我自己有几台 Windows 机器散落在不同地方，每次要上去看一眼都得走一遍完整的远程桌面流程，实在太烦了。后来写着写着就变成了现在这样。

## 三、它能干什么

目前有的功能基本上覆盖了 Windows 远程管理的常见场景，我列一下：

**主机管理**
机器通过 API Key 接进来之后，能看到在线状态、最后心跳时间、IP、系统版本这些基本信息。超时没心跳就自动标离线，不用自己刷半天去猜机器是不是挂了。

**远程终端**
浏览器里直接开 PowerShell 或者 CMD 终端，输入命令就能跑。大部分日常问题其实一条命令就解决了，根本不需要专门去连远程桌面。

**实时屏幕监控**
可以直接看远程机器的屏幕画面，也支持远程控制，鼠标键盘都能操作。如果是临时看个东西修个东西，比开远程桌面快多了。

**文件管理**
浏览远程机器的磁盘目录，上传、下载、删除文件，也支持在远程机器上直接运行 exe。平时排查问题或者传个小文件什么的很方便。

**进程管理**
看远程机器的进程列表，按内存或者 CPU 排序，有异常进程可以直接终止。

**注册表管理**
支持查看和修改远程机器的 Windows 注册表。这个不是天天用，但真要用的时候，不用专门开一次远程桌面就是省事。

**Windows 用户管理**
管理远程机器的本地用户，可以创建、删除、启用、禁用、改密码。日常维护账号的时候用得上。

**日志中心**
各主机的日志会集中收集到平台上，可以按类型、时间、主机筛选。查问题或者做留痕都挺实用。

**API 密钥管理**
管理客户端的接入凭证，支持启用、停用、分类。机器多了之后这一块很重要，不然谁接进来了你都不知道。

**子账号与权限**
可以给不同的人开账号，控制他们能看哪些页面、能管哪些主机。有人只需要看日志，有人需要操作全部机器，都能配。

**KOOK 机器人联动**
如果你们用 KOOK（就是那个国内版的 Discord），这个功能会很实用。平台可以把通知推送到 KOOK 频道里，也能直接从 KOOK 发指令。人在外面的时候，手机上看一眼机器状态，或者执行一条命令，比开电脑快多了。

**AI 助手**
接的是 DeepSeek 的接口，在页面里可以问它某个 Windows 操作该怎么写命令。不是那种很厉害的东西，但偶尔想不起来 PowerShell 语法的时候挺管用的。

**文件投递**
文件上传到服务端之后，可以生成带时效和次数限制的下载链接。适合做临时分发、远程投递之类的场景。

## 四、工作场景

我自己的话，最常用的场景大概这几个：

**管几台 Windows 服务器的团队**
如果你们手上就几台到十几台机器，不想为了这点事上一套又贵又重的平台，这个项目挺合适的。部署门槛不高，打开浏览器就能管。

**远程帮人看问题**
有时候同事或者客户说"我电脑上这个程序打不开了"，其实你只需要看一眼进程、查个目录、跑一条命令。这种时候用这个系统比让人装远程软件快太多了。

**安全巡检**
定期看看各台机器的状态、日志、进程、用户情况，排查一下有没有异常。所有东西都在一个地方，不用东翻西找。

**出门在外应急处理**
这个是我用得最多的场景。人不在电脑边，机器出点小问题，能不能快速看一眼、跑条命令决定要不要回去处理，差别挺大的。

## 五、技术栈

没用什么特别厉害的技术，够用就行。

前端是 WordPress 主题那套东西，PHP 模板加原生 JavaScript，没有 React 也没有 Vue。不是它们不好，是这东西用原生写更直接，少一层构建流程，改起来也方便。

后端就是 WordPress，PHP 8.0+，数据库用 MySQL 或者 MariaDB。

通信走 WordPress REST API 和 Admin AJAX，实时功能走 WebSocket（需要自己搭一个中继服务）。

认证用了 API Key、WebSocket Token 和 WordPress 本身的用户体系，子账号那一套是自己在上面做的。

AI 用的 DeepSeek 的接口。

消息推送接的是 KOOK 的 Bot API。

就这些。没有什么微服务、容器编排、消息队列那些花活，毕竟这个项目解决的问题也没那么复杂。

## 六、关于开源的范围

**说清楚一件事：这个仓库只开源了 Web 管理端，也就是这个 WordPress 主题。**

跑在 Windows 机器上的那个客户端 Agent 是不开源的。Agent 是我自己写的，我打算留着闭源，自己做维护和更新。不是不信任谁，主要是 Agent 这东西涉及到客户端底层的很多东西，我自己管着比较放心。

所以你拿到的就是 Web 端这部分。你可以看、可以用、可以改，但 Agent 那边你得自己想办法。如果你有能力自己写 Agent，那最好不过，接口文档都在代码里，照着调就行。

另外 **本项目的 Web 端禁止商用**。个人用、学习用、内部团队用都没问题。但如果想拿它去做商业项目、卖给别人、或者做收费服务，那不行。要商用的话可以单独联系我。
